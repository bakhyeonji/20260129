## 원본 설계 문서(방향성 참고용)
- `PRD.md`: 제품 방향/기능 후보가 넓게 정리됨 (MVP에선 일부만 사용)
- `IA.md`: 페이지 구조/사용자 플로우가 잘 정리됨 (Day Detail 라우트는 MVP에서 모달로 대체)
- `ERD.md`: 멀티유저/보안까지 고려한 정석 스키마 (MVP에서는 과도하여 축소)
- `DESIGN.md`: UI 톤/컬러/컴포넌트 가이드 (MVP 구현 시 그대로 참고)

## 3시간 MVP에서 “축소”한 핵심 포인트
- **스코프 축소**
  - 유지: 오늘 운세(캐시) + 사진/감정 기록 + 캘린더 회고 + 설정(프로필 수정/초기화)
  - 제외: Auth/멀티유저/LLM/통계/알림/Day Detail 전용 페이지

- **스키마 축소**
  - 원본 ERD(5테이블 + user_id) → MVP(3테이블, date를 PK로 사용)
  - 사진 메타데이터 테이블 제거: `diary_entries.photo_url`에 URL만 저장
